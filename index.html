00

시작 전 핵심 질문

ChatGPT는 대화창에서 질문하는 방식이고, Claude Code는 내 컴퓨터 파일을 직접 읽고 수정하는 AI입니다. 폴더 전체를 이해하고, 코드를 실제로 짜고, 터미널 명령을 실행하고, Git 커밋까지 합니다. "내 컴퓨터에서 일하는 AI 직원"에 가깝습니다.

네. 데스크탑 앱을 쓰면 터미널도 필요 없습니다. 앱 설치 후 폴더를 열고 말하기만 하면 됩니다. "엑셀 파일 정리해줘", "이 사진들 이름 바꿔줘", "웹페이지 만들어줘" 수준으로도 충분히 동작합니다.

2025년부터는 필요 없습니다. 공식 설치 스크립트(curl/PowerShell)나 Homebrew, WinGet으로 네이티브 설치됩니다. npm으로도 여전히 설치할 수 있지만 권장 방식이 아닙니다.

Pro ($20/월), Max 5x ($100/월), Max 20x ($200/월) 구독 플랜이 있습니다. 무료 플랜은 Claude Code를 지원하지 않습니다. API 키로 연결하면 토큰 단위 과금도 가능합니다. 처음엔 Pro로 시작하는 걸 추천합니다.

됩니다. Windows 10 1809 이상을 지원합니다. PowerShell 한 줄, WinGet, 또는 데스크탑 앱으로 설치합니다. WSL2를 쓰면 더 안정적이지만 필수는 아닙니다.

Claude에게 명시적으로 전달한 내용(대화, 파일 내용)만 API를 통해 전송됩니다. Anthropic은 API 데이터로 모델을 학습하지 않습니다. 민감한 파일은 .claudeignore에 등록하면 됩니다.

방법 B — 터미널 스크립트 (개발자)

자동 백그라운드 업데이트가 포함된 공식 CLI 설치 방법이다.

Mac / Linux / WSL 터미널

최신 버전: brew install --cask claude-code@latest

방법 D — WinGet (Windows)

winget install Anthropic.ClaudeCode

방법 E — Linux 패키지 관리자

sudo install -d -m 0755 /etc/apt/keyrings
sudo curl -fsSL https://downloads.claude.ai/keys/claude-code.asc \
  -o /etc/apt/keyrings/claude-code.asc
echo "deb [signed-by=/etc/apt/keyrings/claude-code.asc] \
  https://downloads.claude.ai/claude-code/apt/stable stable main" \
  | sudo tee /etc/apt/sources.list.d/claude-code.list
sudo apt update && sudo apt install claude-code
sudo tee /etc/yum.repos.d/claude-code.repo <<'EOF'
[claude-code]
name=Claude Code
baseurl=https://downloads.claude.ai/claude-code/rpm/stable
enabled=1
gpgcheck=1
gpgkey=https://downloads.claude.ai/keys/claude-code.asc
EOF
sudo dnf install claude-code
wget -O /etc/apk/keys/claude-code.rsa.pub \
  https://downloads.claude.ai/keys/claude-code.rsa.pub
echo "https://downloads.claude.ai/claude-code/apk/stable" >> /etc/apk/repositories
apk add claude-code

05

CLAUDE.md — 나만의 작업 규칙

CLAUDE.md는 Claude가 세션 시작 시 자동으로 읽는 규칙 파일이다. "항상 한국어로 답해", "변수명은 camelCase", "테스트 꼭 작성" 같은 규칙을 한 번 써두면 매번 말할 필요가 없다.

파일 위치 (범위별)

위치범위공유
~/.claude/CLAUDE.md내 모든 프로젝트본인만
./CLAUDE.md 또는 ./.claude/CLAUDE.md해당 프로젝트팀원 (Git 커밋)
./CLAUDE.local.md해당 프로젝트 (개인)본인만 (.gitignore에 추가)
💡 /init 으로 자동 생성

대화창에서 /init을 실행하면 Claude가 코드베이스를 분석해서 CLAUDE.md를 자동으로 만들어준다. 이미 있으면 개선 사항을 제안한다.

CLAUDE.md 예시

./CLAUDE.md
# 팀 규칙

## 언어
- 모든 답변은 한국어로
- 기술 용어·에러메시지는 영어 그대로

## 코딩 스타일
- 들여쓰기: 2칸 공백
- 변수명: camelCase
- 세미콜론 사용 (TypeScript)

## 금지 사항
- API 키 하드코딩 절대 금지 (.env 파일 사용)
- main 브랜치 직접 커밋 금지

## 커밋 규칙
형식: type(scope): 설명
예: feat(auth): 소셜 로그인 추가

.claude/rules/ — 규칙 분리 관리

프로젝트가 커지면 .claude/rules/ 폴더에 주제별로 나눠서 관리한다.

.claude/
└── rules/
    ├── code-style.md    # 코딩 스타일 규칙
    ├── security.md      # 보안 규칙
    └── testing.md       # 테스트 규칙

경로별 규칙 (특정 폴더에만 적용)

.claude/rules/frontend.md
---
paths:
  - "src/components/**"
  - "src/pages/**"
---

# 프론트엔드 규칙
- React 함수 컴포넌트만 사용
- Props는 TypeScript interface로 정의
- CSS-in-JS 대신 Tailwind 사용

@import로 파일 불러오기

CLAUDE.md
@README              # README 파일을 컨텍스트로 포함
@package.json        # 패키지 정보 포함

# 추가 규칙
@.claude/rules/code-style.md
⚠️ CLAUDE.md 작성 팁

파일당 200줄 이내를 권장한다. 길수록 Claude가 중요 규칙을 놓칠 수 있다. HTML 주석(<!-- -->)은 자동 제거되어 무시된다. 규칙은 검증 가능하게 구체적으로 써야 효과가 있다. ("코드를 잘 포맷합니다" ❌ → "2칸 들여쓰기 사용" ✅)

06

Auto-Memory — 세션을 넘는 기억

NEW v2.1.59+

Claude Code v2.1.59부터 추가된 기능이다. Claude가 대화하면서 스스로 기억할 것을 판단하고 저장한다. 다음 세션에서도 이전 맥락이 자동으로 이어진다.

✦ Auto-Memory vs CLAUDE.md 차이
CLAUDE.mdAuto-Memory
작성자사용자Claude 자동 저장
내용지침 & 규칙학습 & 패턴
예시코딩 표준, 금지 사항빌드 명령, 디버깅 인사이트

메모리 파일 위치

~/.claude/projects/<프로젝트경로>/memory/
├── MEMORY.md        # 인덱스 (모든 세션에 처음 200줄 로드)
├── debugging.md     # 디버깅 패턴 (필요시 로드)
└── api-conventions.md  # API 설계 결정 (필요시 로드)

MEMORY.md의 처음 200줄 또는 25KB가 세션 시작 시 자동 로드된다. 상세 내용은 별도 주제 파일에 저장된다.

메모리 관리

대화 또는 명령어
/memory                          # 메모리 파일 보기 · 편집
> 이 프로젝트는 Python 3.11을 쓴다고 기억해줘
> pnpm이 아닌 npm을 쓴다고 기억해줘
> 내 이름은 재연이야, 기억해줘

비활성화 방법

# settings.json에서
{
  "autoMemoryEnabled": false
}

# 환경 변수로
CLAUDE_CODE_DISABLE_AUTO_MEMORY=1 claude

저장 위치 변경

~/.claude/settings.json
{
  "autoMemoryDirectory": "~/my-memory-dir"
}
07

Skills — 자동 호출되는 나만의 명령

핵심 기능

스킬(Skills)은 내가 만드는 커스텀 명령이다. description을 읽고 Claude가 상황에 맞게 자동으로 호출한다. /스킬명으로 직접 부르는 것도 가능하다.

✦ 달라진 점

기존 .claude/commands/ 파일 방식도 여전히 작동하지만, Skills 방식이 권장된다. 자동 호출, 지원 파일, YAML frontmatter로 더 강력하게 사용할 수 있다.

스킬 위치

위치적용 범위
~/.claude/skills/<이름>/SKILL.md내 모든 프로젝트
.claude/skills/<이름>/SKILL.md해당 프로젝트만

첫 번째 스킬 만들기

터미널
mkdir -p ~/.claude/skills/daily-summary
~/.claude/skills/daily-summary/SKILL.md
---
description: 오늘 작업한 내용을 요약할 때. "오늘 뭐 했어", "daily summary", "작업 요약" 등에 자동 호출.
---

# 일일 작업 요약

git log --since="today 00:00" --oneline 으로 오늘 커밋을 조회하고,
다음 형식으로 보고서를 작성한다:

## 일일 보고서 (오늘 날짜)
### 완료한 작업
### 이슈 / 블로커
### 내일 할 일

YAML Frontmatter 주요 필드

필드설명
name스킬 이름 (생략 시 디렉토리명 사용)
description자동 호출 판단 기준. 구체적일수록 정확함 (권장)
when_to_usedescription에 추가되는 트리거 상황 설명
disable-model-invocationtrue → 자동 호출 차단, 수동 /스킬명만 가능
user-invocablefalse → / 메뉴에서 숨김 (Claude만 호출 가능)
allowed-tools승인 없이 사용할 수 있는 도구 목록
contextfork → 격리된 subagent에서 실행
argument-hint자동완성 시 표시되는 인수 힌트 (예: [파일명])
paths특정 파일 패턴에서만 자동 활성화

인수 전달 & 동적 컨텍스트

인수 예시 — /fix-issue 123
---
name: fix-issue
description: GitHub 이슈 번호로 버그 수정
disable-model-invocation: true
---

GitHub 이슈 $ARGUMENTS 를 우리 코딩 표준에 맞게 수정한다.

1. 이슈 내용 읽기
2. 원인 파악
3. 수정 구현
4. 테스트 작성
5. 커밋 생성
동적 컨텍스트 주입 — !`명령어`
---
description: 현재 변경사항 요약 및 위험 요소 표시
---

## 현재 변경사항

!`git diff HEAD`

## 지시사항

위 변경사항을 2~3 bullet로 요약하고,
누락된 에러 처리, 하드코딩 값, 업데이트가 필요한 테스트 등 위험 요소를 나열하라.

`` !`명령어` `` 는 Claude가 보기 전에 미리 실행되어 출력으로 대체된다.

💡 description 잘 쓰는 법

description은 Claude가 "언제 쓸지" 판단하는 유일한 단서다. 트리거 단어를 구체적으로 열거하면 정확도가 높아진다.
좋은 예: "사용자가 '배포', 'deploy', '서버에 올려' 등을 말할 때"
나쁜 예: "코드 관련 작업" (너무 광범위)

Hooks — 이벤트 자동화

훅(Hooks)은 특정 이벤트 발생 시 자동 실행되는 스크립트다. 파일 수정 시 lint 자동 실행, Claude 명령 차단, Slack 알림 전송 등 강력한 자동화가 가능하다.

주요 이벤트

이벤트언제
PreToolUse도구(파일 수정, 명령 실행 등) 실행 직전 — 차단 가능
PostToolUse도구 실행 직후
UserPromptSubmit내가 메시지를 전송할 때
SessionStart세션 시작 시
SessionEnd세션 종료 시
FileChanged감시 파일이 변경될 때
CwdChanged작업 디렉토리가 변경될 때
ConfigChange설정 파일 변경 시
PermissionRequestClaude가 권한을 요청할 때
StopClaude 응답 완료 시
InstructionsLoadedCLAUDE.md 또는 규칙 파일 로드 시
PreCompact / PostCompact컨텍스트 압축 전/후

훅 타입 5가지

타입설명
command셸 명령어 실행 (기본)
httpHTTP POST 요청 전송 (웹훅, Slack 등)

agent다른 Claude 에이전트 호출

설정 방법 — /hooks 메뉴

대화창에서 /hooks를 입력하면 그래픽 설정 메뉴가 열린다. 코드를 직접 안 써도 된다.

설정 파일로 직접 작성

.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npx",
            "args": ["eslint", "${tool_input.file_path}", "--fix"],
            "timeout": 30
          }
        ]
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash",
        "if": "Bash(rm -rf *)",
        "hooks": [
          {
            "type": "command",
            "command": "echo",
            "args": ["rm -rf 명령은 차단됩니다"],
            "timeout": 5
          }
        ]
      }
    ]
  }
}
⚠️ 주의

훅은 강력하지만 잘못 설정하면 무한 루프나 의도치 않은 동작을 유발할 수 있다. 처음엔 echo나 로그 기록 같은 안전한 명령부터 시작하자.

MCP 서버 — 능력 확장

MCP(Model Context Protocol)는 Claude에게 외부 도구를 연결하는 표준 방식이다. MCP 서버를 추가하면 Claude가 데이터베이스 조회, 웹 검색, Slack 메시지 전송, GitHub 조작 등을 직접 할 수 있다.

MCP 추가 — /mcp 메뉴

/mcp    # 대화창에서 입력 → 서버 검색 및 추가 메뉴

설정 파일로 직접 추가

~/.claude/settings.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/나/Documents"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_xxxx" }
    }
  }
}
⚠️ 보안

MCP 서버는 외부 코드를 실행한다. 신뢰할 수 있는 출처의 서버만 추가하고, GITHUB_TOKEN 같은 민감 정보는 환경 변수로 관리한다.

@modelcontextprotocol/server-brave-search웹 검색
@modelcontextprotocol/server-slackSlack 메시지 전송/조회
@modelcontextprotocol/server-google-driveGoogle Drive 파일 접근

트러블슈팅

# 재설치
curl -fsSL https://claude.ai/install.sh | bash

# 설치 후 터미널 재시작, 그래도 안 되면:
source ~/.bashrc   # 또는 source ~/.zshrc

# 진단 실행
claude doctor
claude auth logout
claude auth login

그래도 안 되면 ~/.claude/ 폴더와 ~/.claude.json을 삭제 후 재시작한다.

/compact로 대화를 압축하거나 /clear 후 재시작한다. 컨텍스트가 너무 길면 속도가 느려지고 응답이 잘릴 수 있다.

오류 메시지에 표시된 시간만큼 기다리거나 Max 플랜으로 업그레이드한다. 또는 /upgrade로 플랜 변경 페이지로 이동할 수 있다.

/memory를 실행해서 파일이 로드되었는지 확인한다. 규칙이 너무 모호하면 Claude가 따르지 않을 수 있다. 더 구체적으로 ("2칸 들여쓰기 사용") 작성하고, 충돌하는 규칙이 없는지 확인한다.

description 필드를 더 구체적으로 작성한다. disable-model-invocation: true가 설정되어 있으면 자동 호출이 비활성화된 상태다. /skills로 스킬이 인식되는지 확인한다.

pwd로 현재 위치를 확인하고 올바른 폴더에서 claude를 실행했는지 확인한다. claude doctor로 전반적인 설정 상태도 점검한다.

팀 협업

팀 공용 CLAUDE.md

프로젝트 루트의 CLAUDE.md를 Git에 커밋하면 팀원 모두가 같은 규칙으로 Claude를 사용한다.

💡 개인 규칙 + 팀 규칙 분리

~/.claude/CLAUDE.md에 개인 취향(응답 언어, 설명 방식 등)을 넣고, 팀 공용 규칙은 프로젝트 CLAUDE.md에 넣는다. 두 파일은 자동으로 합쳐진다.

팀 공용 스킬

.claude/skills/를 Git에 커밋하면 팀 전체가 같은 스킬을 사용할 수 있다.

.claude/
├── CLAUDE.md            # 팀 규칙 (Git 커밋)
└── skills/
    ├── deploy/SKILL.md  # 팀 배포 스킬 (Git 커밋)

Claude Code SDK — 프로그램에서 호출

Python SDK
import anthropic

client = anthropic.Anthropic()

with open("report.txt") as f:
    content = f.read()

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    messages=[{
        "role": "user",
        "content": f"다음 보고서를 3줄로 요약해줘:\n\n{content}"
    }]
)
print(response.content[0].text)

CI/CD 파이프라인 연동

.github/workflows/claude-review.yml
name: Claude Code Review
on: [pull_request]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Claude 코드 리뷰
        run: |
          git diff origin/main...HEAD > changes.diff
          cat changes.diff | claude -p "이 변경사항에서 버그나 보안 이슈를 찾아줘"
